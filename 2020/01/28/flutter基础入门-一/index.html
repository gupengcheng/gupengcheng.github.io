    • flutter基础入门(一)

    flutter从第一次接触到现在也有快2年了，从最开始可以跨平台iOS、Android，到后来1.0版本发布，官方不断完善，到可以支持wb，未来可能还会支持windows、macOS等，成为真正的跨平台开发工具，所以准备系统的总结一下flutter从入门开始的一些内容。

    1、flutter是什么

    看一下官方是怎么介绍flutter的：

    Flutter is Google’s UI toolkit for building beautiful, natively compiled applications for mobile, wb, and desktop from a single codebase.

    flutter其实是google开发的一套跨平台UI框架，他和react native的区别在于，flutter不是通过编译转化为原生语言的方式来实现，而是通过自己实现的一套绘制框架来实现直接将dart语言编写的widget绘制在iOS、Android设备上实现跨平台。

    flutter的优点有了，一套代码，编译后多端运行，并且不是通过转换为原生控件的方式，而是自研的绘制框架。但是缺点也很明显，所有的实现都是通过类似于wb react语言的框架，阅读较普通的客户端开发会更难，并且代码会越写越长，难于维护，另外，当前flutter的生态还不够，很多框架还没有现成可以用的。

    2、flutter中最重要的概念是什么，毋庸置疑是widget组件的概念

    flutter中所有内容都是组件widget，一个页面是一个widget，一个按钮是一个widget，widget里面还可以包含其他的widget，这样你可以通过组合实现很复杂的页面。

    3、dart和flutter的关系

    flutter的官方开发语言就是dart语言，dart是google自己研究的一门类似于wb开发的语言，最初也是用于wb开发，一直不温不火，后来终于等到了flutter将dart语言一起带火了起来。

    所以，要想学习flutter，首先，你要熟悉一下dart的基本语法。当然，如果你又java的基础，其实是很简单的。

    4、flutter程序中入口是什么

    新建一个flutter application后，在lib下面会生成一个main.dart文件，这就是类似于Android程序中的MainActivity，入口方法即是其中的main方法，对应的方法内容是runApp()，runApp需要传递一个Widget给他，即这里的MyApp就是一个Widget。

    5、widget的种类有两种，有状态widget(StatefulWidget)和无状态widget(StatelessWidget)

    这里简单的说一下这两种widget的区别：

    1).StatelessWidget用于不需要状态改变的widget，比如申明一段文字的widget，其实也可以认为是不需要和用户交互的一种widget。

    2).StatefulWidget是一种支持改变状态的Widget，不如一个button，原来是灰色背景的确定按钮，点击后需要变成红色背景的取消按钮，这就需要实现一个StatefulWidget。

    实现一个StatefulWidget(比如MyHomePage)需要两步，首先需要继承一个StatefulWidget并实现createState方法返回一个State，这里没有State，那么你就还需要继承一个State并实现build方法返回一个Widget，然后就可以将这里的State赋给上面的createState方法的返回值。
