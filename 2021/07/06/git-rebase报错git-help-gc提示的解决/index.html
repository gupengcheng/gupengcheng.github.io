git rebase报错git help gc提示的解决

Posted on 2021-07-06

某天项目git rebase报错如上，提示为：

  1. 自动包装最佳性能的存储库。
  2. 你也可以手动运行 ‘get gc’ 。查看更多信息请执行“git help gc”

产生的原因是：

1、Git 往磁盘保存对象时默认使用的格式叫松散对象 (loose object) 格式。Git 时不时地将这些对象打包至一个叫 packfile 的二进制文件以节省空间并提高效率。当仓库中有太多的松散对象则就会提示你运行 ’ git gc ‘。

2、自己本地一些 “悬空对象”太多(git删除分支或者清空stash的时候，这些其实还没有真正删除，成为悬空对象，我们可以使用merge命令可以从中恢复一些文件)

解决方式：

1、运行命令查看有哪些悬空对象git fsck --lost-found

2、执行命令git gc --prune=now删除这些对象，但是这里执行后可能继续报错如下：

这时候需要把.git/refs/remotes/m/git-ssh重命名到一个tmp目录

再次执行上述2命令，即可。
